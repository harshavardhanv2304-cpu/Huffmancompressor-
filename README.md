# Huffman File Compressor

A **lossless text compression utility** built using the greedy Huffman Coding algorithm and object-oriented C++ principles.

---

## Features

- **Greedy Huffman Coding** — builds an optimal prefix tree using character frequencies
- **OOP Design** — all logic encapsulated in a modular `Huffman` class
- **Priority Queue + STL Map** — `priority_queue` for min-heap tree construction, `map` for frequency tables and code lookup
- **Bit-level I/O Serialization** — variable-length prefix codes packed into 8-bit binary streams
- **Custom Binary File Header** — stores frequency table and bit-padding metadata for guaranteed 100% lossless text recovery
- **Zero Memory Leaks** — recursive destructor cleans up the entire Huffman tree

**Requirements:** g++ with C++17 support

```bash
# Build
make

# Compress a file
./huffman compress input.txt output.huf

# Decompress
./huffman decompress output.huf recovered.txt

make test
```

## How It Works

### Compression Pipeline

1. **Frequency Table** — scan input file and count character occurrences using `std::map<char, int>`
2. **Prefix Tree** — insert all characters as leaf nodes into a `priority_queue` (min-heap); repeatedly merge the two lowest-frequency nodes until one root remains
3. **Code Map** — traverse the tree recursively; left edge = `0`, right edge = `1`; store each character's prefix code in a `std::map<char, string>`
4. **Encode** — replace every character in the file with its variable-length prefix code
5. **Bit Packing** — pack the bit string into bytes (8-bit buffer packing), recording padding needed to align to byte boundary
6. **Binary Header** — write: `[numChars(4B)] [char(1B) freq(4B)] × N [padding(1B)]` before compressed data
7. **Write Output** — header + packed binary data → `.huf` file

### Decompression Pipeline

1. **Read Header** — parse frequency table and padding from binary header
2. **Rebuild Tree** — reconstruct the exact same Huffman tree from the frequency table
3. **Read Bits** — unpack bytes back to bit string, strip padding
4. **Decode** — walk the tree bit-by-bit; emit a character at each leaf node

## Binary File Format

```
┌─────────────────────────────────────┐
│  numEntries  (4 bytes, int)         │
├─────────────────────────────────────┤
│  char (1B) + frequency (4B)  × N   │  ← frequency table
├─────────────────────────────────────┤
│  padding (1 byte)                   │  ← bit-pad metadata
├─────────────────────────────────────┤
│  compressed binary data             │
└─────────────────────────────────────┘
```
